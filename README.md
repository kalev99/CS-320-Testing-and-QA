# CS-320-Testing-and-QA
This repository contains part of a project demonstrating skill with coding and simple testing

# How can I ensure that my code, program, or software is functional and secure?
# How do I interpret user needs and incorporate them into a program?
# How do I approach designing software?

I like to joke that my life motto is "Carefully and with great caution", as that is how I approach most aspects of life. My approach to designing software is the same - each step should be planned and double-checked to avoid errors in the long run. I do this by examining the requirements or user needs and breaking them down into logic branches that I could code. By testing, I proved that the code I created was functional and secure. By examining the testing coverage, I could locate where my tests were implemented incorrectly, allowing me to debug the test files as well.
